# CLAUDE.md — 반려동물 사주 프로젝트 공통 지침

> 이 문서는 AI(Claude)가 이 프로젝트를 작업할 때 참고하는 "작업 매뉴얼"입니다.

## 1. 이 프로젝트가 뭔가요?

**반려동물 사주 웹사이트**입니다. 사용자가 반려동물의 생년월일을 입력하면, 전통 사주(오행)를 재미있게 변형해서 성격 분석과 운세를 보여주는 **엔터테인먼트 서비스**입니다.

핵심 컨셉을 한 줄로 요약하면:
> "생년월일만 넣으면, 우리 아이의 타고난 운명을 귀엽고 재미있게 알려준다"

지원 동물: 강아지 🐶, 고양이 🐱, 햄스터 🐹, 토끼 🐰, 새 🐦, 파충류 🦎

GitHub: https://github.com/happyjiwoodayy-soapy/pet-saju
배포: Vercel (정적 호스팅)

## 2. 폴더 구조와 각 파일의 역할

```
pet-saju/
├── index.html          ← 유일한 HTML 파일. 4개 화면이 모두 이 안에 있음
├── css/
│   └── style.css       ← 모든 디자인(색상, 레이아웃, 애니메이션)
├── js/
│   ├── saju-engine.js  ← 사주 분석 엔진 (천간지지 계산, 오행 분포, 성격 유형 판별)
│   ├── app.js          ← 화면 전환, 입력 폼 처리, 결과 표시 등 UI 로직
│   └── share.js        ← 공유(카카오톡, 링크 복사)와 결과 이미지 저장
└── CLAUDE.md           ← 이 파일 (AI 작업 지침)
```

### 쉽게 비유하면
- `index.html` = 건물의 뼈대 (구조)
- `style.css` = 인테리어 (예쁘게 꾸미기)
- `saju-engine.js` = 두뇌 (사주를 계산하는 핵심 로직)
- `app.js` = 안내 데스크 (사용자의 클릭에 반응, 화면 이동)
- `share.js` = 홍보팀 (결과를 친구에게 공유)

## 3. 기술 스택 (이 프로젝트에서 쓰는 도구들)

| 구분 | 사용 기술 | 쉬운 설명 |
|------|-----------|-----------|
| 언어 | HTML + CSS + Vanilla JavaScript | 프레임워크(React 등) 없이 순수 웹 기술만 사용 |
| 폰트 | Pretendard Variable | 한국어가 깔끔하게 보이는 무료 폰트 |
| 이미지 저장 | html2canvas | 화면을 사진처럼 찍어 저장하는 라이브러리 |
| 공유 | 카카오 SDK, Web Share API | 카카오톡 공유, 모바일 기본 공유 기능 |
| 호스팅 | Vercel | 코드를 GitHub에 올리면 자동으로 웹에 배포 |

**중요한 원칙:**
- 서버가 없습니다. 모든 계산은 사용자의 브라우저에서 이루어집니다.
- 외부 API 호출이 없습니다. 인터넷이 끊겨도 한번 로딩되면 동작합니다.
- 사용자의 개인정보를 서버에 저장하지 않습니다.

## 4. 화면 구성 (4개 화면, 한 페이지 안에서 전환)

이 사이트는 하나의 HTML 페이지 안에 4개 화면(section)이 들어있고, JavaScript가 화면을 전환합니다.

### 화면 1: 랜딩 (첫 화면)
- 귀여운 동물 이모지가 떠다니는 애니메이션
- "사주 보러 가기" 버튼 하나

### 화면 2: 정보 입력
- 동물 종류 선택 (6종)
- 이름, 생년월일, 성별 입력
- 선택사항: 태어난 시간, 보호자 생년월일(궁합용)

### 화면 3: 로딩 애니메이션
- 프로그레스 바 + 재미있는 로딩 메시지
- 실제 계산은 즉시 끝나지만, 기대감을 위해 약 3초 지연

### 화면 4: 결과 페이지
- 사주 캐릭터 카드 (유형 이름 + 이모지 + 한 줄 소개)
- 오행 분석 막대 그래프 (목/화/토/금/수)
- 성격 별점 (활발함, 고집, 애교, 지능, 사교성)
- 숨겨진 매력 텍스트 (사주 데이터 기반 동적 생성)
- 보호자 궁합 (입력한 경우)
- 행운의 아이템 (색상, 간식, 산책 시간, 베스트 프렌드)
- 이미지 저장 / SNS 공유 버튼

## 5. 사주 엔진 — 핵심 로직 설명

### 쉬운 비유
사주 엔진은 "생년월일 → 운명 해석기"입니다. 날짜를 넣으면 오행(목·화·토·금·수)의 비율을 계산하고, 그 비율에 따라 성격 유형을 정해줍니다.

### 계산 흐름
```
입력(생년월일시) 
  → 천간·지지 4주(연주/월주/일주/시주) 산출
  → 지지장간(숨은 천간)까지 포함해 오행 점수 합산
  → 동물별 미세 보정 적용 (+0.5 수준)
  → 오행 백분율 계산
  → 상위 2개 오행 조합으로 16가지 유형 중 매칭
  → 성격 별점, 숨겨진 매력, 행운 정보 생성
```

### 핵심 데이터 구조

**천간(天干):** 갑, 을, 병, 정, 무, 기, 경, 신, 임, 계 (10개)
**지지(地支):** 자, 축, 인, 묘, 진, 사, 오, 미, 신, 유, 술, 해 (12개)
**오행:** 목(나무), 화(불), 토(흙), 금(쇠), 수(물)

**지지장간:** 각 지지 안에 숨어있는 천간. 예: 인(寅) = 갑목(본기 1.0) + 병화(중기 0.6) + 무토(여기 0.3). 이걸 반영해야 결과가 다양해집니다.

**성격 유형:** 16가지 (오행 상위 2개 조합). 예: "불꽃 리더" (화+목), "신비로운 몽상가" (수+목) 등

### 결과 다양성을 위한 장치
- 지지장간 가중치 반영 (단순히 지지=오행 1:1 매핑하면 결과가 편향됨)
- 동물 보정 가중치는 +0.5로 작게 유지 (날짜가 결과를 주도해야 함)
- 이름 해시 기반으로 성격 별점에 -1~+1 변동 부여
- 숨겨진 매력은 일간, 오행 분포, 동물 종류를 조합한 5개 층위 동적 생성

## 7. 코딩 규칙

### 전체 규칙
- **한글 변수명 사용**: 사주 관련 데이터는 한글 변수명을 씁니다. `const 천간 = [...]`, `const 지지오행 = {...}`
- **영어 변수명**: UI 로직이나 일반적인 프로그래밍 요소는 영어. `state`, `screens`, `showToast()`
- **IIFE 패턴**: 각 JS 파일은 즉시실행함수 또는 모듈 패턴으로 감싸서 전역 오염 방지
  - `saju-engine.js` → `const SajuEngine = (() => { ... })()`
  - `app.js` → `(function() { ... })()`
  - `share.js` → `const ShareManager = (() => { ... })()`
- **'use strict'** 사용
- **세미콜론** 사용
- 들여쓰기: 스페이스 2칸

### CSS 규칙
- CSS 변수(custom properties) 적극 활용 (`:root`에 정의)
- BEM은 안 쓰되, 화면별 접두사로 구분: `.landing-`, `.input-`, `.loading-`, `.result-`
- 미디어 쿼리는 모바일 퍼스트: `min-width`로 작성

### HTML 규칙
- 시맨틱 태그 사용: `<section>`, `<main>`, `<header>`, `<button>`
- 각 화면은 `<section id="screen-xxx" class="screen">` 형태
- 활성 화면은 `.active` 클래스로 표시

## 8. 작업 시 주의사항

### 절대 하지 말 것
1. **프레임워크 도입 금지** — React, Vue, Svelte 등을 추가하지 마세요. 순수 HTML/CSS/JS를 유지합니다.
2. **서버 사이드 로직 추가 금지** — Node.js 서버, API 엔드포인트 등을 만들지 마세요. 100% 클라이언트 사이드입니다.
3. **사주 엔진의 지지장간 제거 금지** — 지지장간 가중치를 빼면 결과 다양성이 급감합니다. 반드시 유지하세요.
4. **동물 오행 보정을 크게 올리지 말 것** — 현재 +0.5 수준. 이걸 +2 이상으로 올리면 같은 동물은 비슷한 결과만 나옵니다.
5. **결과 텍스트를 하드코딩하지 말 것** — 숨겨진 매력 등의 텍스트는 동적 생성 시스템을 통해 만들어야 합니다.

### 꼭 지킬 것
1. **결과 다양성 확인** — 사주 엔진을 수정하면 반드시 여러 입력(다른 동물, 다른 날짜 10개 이상)으로 테스트해서 결과가 충분히 다양한지 확인하세요.
2. **모바일 테스트 우선** — 이 서비스의 주요 사용자는 모바일입니다. PC보다 모바일에서의 동작을 먼저 확인하세요.
3. **한국어 조사 처리** — "은/는", "이/가", "을/를" 등 한국어 조사가 자연스러운지 확인하세요. 받침 유무에 따라 달라집니다.
4. **이모지 호환성** — 사용하는 이모지가 주요 OS(iOS, Android)에서 모두 보이는지 확인하세요.
5. **성능 목표** — 첫 화면(LCP) 2.5초 이내. 외부 리소스 최소화.

## 10. 자주 하는 작업 가이드

### 새 성격 유형 추가하기
1. `saju-engine.js`의 `유형데이터` 배열에 새 유형 추가
2. 오행 조합 키(예: `'화목'`)를 지정
3. `name`, `emoji`, `subtitle`, `stats` 정의
4. 숨겨진 매력 동적 생성기가 자동으로 새 유형에 맞는 텍스트를 만들어줌
5. 테스트: 해당 유형이 실제로 나오는 입력값을 찾아 확인

### 새 동물 종류 추가하기
1. `saju-engine.js`의 `동물이모지`, `동물이름`, `동물오행보정` 에 추가
2. 동물별 행운 정보(`행운데이터`)에 새 동물용 데이터 추가
3. 숨겨진 매력의 `동물행동` 텍스트에 새 동물용 행동 묘사 추가
4. `index.html`의 동물 선택 버튼에 새 아이콘 추가
5. `style.css`에서 필요시 동물별 스타일 추가

### 디자인 수정하기
1. 색상 변경: `style.css`의 `:root` CSS 변수만 수정하면 전체에 적용
2. 간격 변경: `--space-xs` ~ `--space-2xl` 변수 활용
3. 새 애니메이션: `@keyframes` 정의 후 `transform`/`opacity` 위주로 작성

### 배포하기
1. 코드를 수정하고 GitHub에 push하면 Vercel이 자동 배포
2. `git add .` → `git commit -m "변경 설명"` → `git push origin main`
3. 1~2분 후 자동 반영

## 11. 비개발자를 위한 용어 사전

| 용어 | 쉬운 설명 |
|------|-----------|
| HTML | 웹페이지의 뼈대. "여기에 버튼 넣고, 여기에 글자 넣고" 하는 구조 |
| CSS | 웹페이지의 화장. 색상, 크기, 위치, 애니메이션 등 꾸미기 담당 |
| JavaScript (JS) | 웹페이지의 두뇌. 클릭하면 반응하고, 계산하고, 화면 바꾸는 역할 |

| 프레임워크 | 개발을 편하게 해주는 도구 세트 (React, Vue 등). 이 프로젝트에선 안 씀 |
| IIFE | 파일이 로드되자마자 바로 실행되는 함수 묶음. 코드끼리 충돌 방지용 |
| 천간/지지 | 전통 사주에서 쓰는 기호. 천간 10개 + 지지 12개 = 날짜를 표현하는 체계 |
| 오행 | 목(나무), 화(불), 토(흙), 금(쇠), 수(물). 사주의 핵심 다섯 가지 기운 |
| 지지장간 | 지지 속에 숨어있는 추가 기운. 이걸 계산해야 결과가 다양해짐 |
| 일간 | 사주에서 "나 자신"을 뜻하는 천간. 생일의 천간이 곧 일간 |
| 클라이언트 사이드 | 서버가 아닌 사용자의 브라우저에서 실행된다는 뜻 |
| MVP | Minimum Viable Product. 핵심 기능만 갖춘 최소 버전 |
| LCP | 화면에 가장 큰 요소가 표시되는 시간. 빠를수록 좋음 (목표: 2.5초 이내) |
| Vercel | 웹사이트를 인터넷에 올려주는 서비스. GitHub에 코드 올리면 자동 배포 |
| 해시 | 어떤 값을 규칙에 따라 숫자로 바꾸는 것. 같은 이름이면 항상 같은 숫자 |
| GPU 가속 | 컴퓨터의 그래픽 칩을 활용해 애니메이션을 부드럽게 만드는 기법 |
| safe area | 아이폰 노치(화면 상단 홈)나 하단 바가 가리는 영역을 피하는 것 |

## 12. 이 문서를 활용하는 방법

Claude(AI)가 이 프로젝트 관련 작업을 할 때:

1. **코드를 수정하기 전에** 이 문서의 "코딩 규칙"과 "주의사항"을 먼저 확인하세요.
2. **사주 엔진을 건드릴 때** "사주 엔진" 섹션을 반드시 읽고, 결과 다양성 테스트를 꼭 하세요.
3. **새 기능을 추가할 때** "기능 확장 로드맵"을 참고해서 우선순위에 맞는 작업인지 확인하세요.
4. **디자인을 바꿀 때** "디자인 시스템" 섹션의 색상 팔레트와 원칙을 따르세요.
5. **비개발자에게 설명할 때** "용어 사전"을 활용해 쉬운 말로 설명하세요.
